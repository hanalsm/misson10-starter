# /shield

현재 브랜치의 변경된 파일을 대상으로 코드 품질을 점검합니다.

## 동작
변경된 파일에서 아래 항목을 순서대로 검토하고 결과를 보고합니다.

1. **타입 안전성** — `any` 타입 사용 위치 리스트업
2. **Server / Client 경계** — 불필요한 `use client` 감지 (Server Component로 전환 가능한 경우 안내)
3. **스타일 일관성** — Tailwind 클래스 중복, 인라인 style 혼용 여부
4. **디버그 코드** — 남겨진 `console.log` 위치
5. **종합 의견** — 심각도별 (🔴 필수 / 🟡 권장 / 🟢 참고) 정리

## 규칙
- `git diff --name-only` 기준으로 변경 파일만 검토
- 문제가 없으면 "이상 없음"으로 간결하게 보고
- 수정이 필요한 경우 파일명:라인번호 형식으로 위치 안내
